# Implementation Notes — Issue #94

LLM (Claude) の思考言語を英語ベースに統一する方針を、`CLAUDE.md` 本体（root および
`repo-template/`）に h2 節として明文化した実装の補足ノート。

## 設計判断

本 Issue は Architect なしフロー（ドキュメント変更のみ、設計判断は粒度小）として進めた。
Open Questions に対する判断と根拠を以下に記録する。

### 1. 配置場所: CLAUDE.md 本体 vs 独立ファイル

**判断**: CLAUDE.md 本体に h2 節として追記する（独立 `.claude/rules/language-policy.md` には
切り出さない）。

**根拠**:

- CLAUDE.md 冒頭の「すべてのエージェントは作業開始前にこのファイルを読み直してください」
  というスコープに自動的に含まれ、Req 5.1（到達性）を最小コストで満たせる
- 独立ファイルにすると個別エージェントの Read 漏れリスクが発生し、Req 5.3（個別書き換え
  不要で方針が届く）の検証が難しくなる
- 既存の `## 禁止事項` / `## エージェント連携ルール` 等も CLAUDE.md 本体内で完結している
  運用上の慣習と整合する
- 行数は 38 行で NFR 1.1（60 行以内）に十分収まり、本体追記による可読性悪化は最小限

### 2. 配置位置（h2 順序）

**判断**: `## このリポジトリについて`（root のみ）またはイントロの直後、`## 技術スタック`
の前に挿入。

**根拠**:

- 言語選択は「コード規約」「ブランチ・コミット規約」など複数の既存節を横断する原則のため、
  それらの節より前に配置することで参照される順序が自然になる
- 既存 h2 節の **順序・名前・本文は変更していない**（Req 3.1 / 3.2 の後方互換性要件を充足）

### 3. コミットメッセージ本文の言語

**判断**: 「日本語ベース」と分類。

**根拠**:

- `git log -5 --format='%s%n%n%b'` で最近のコミットを確認したところ、subject は英語の
  Conventional Commits prefix、body は日本語が主流（例: `fix(watcher): drop inline git
  diff...` の body が「Reviewer プロンプトに git diff... の inline 抜粋を撤廃し...」と
  日本語で書かれている）
- 既存運用に倣う形で本方針を成文化することで、運用上の差し戻しが発生しない

### 4. bash ログ出力メッセージ

**判断**: 「混在許容」と分類。新規追加分は日本語ベースを推奨するが、既存実装の書き換え
は本 Issue のスコープ外（requirements.md の Out of Scope と整合）。

**根拠**:

- `local-watcher/bin/issue-watcher.sh` の既存 `echo` 文字列は日本語混在の現状（例:
  「処理対象の Issue なし」「Triage failed」）。一律日本語化／英語化は別 Issue 案件
- 本方針で「混在許容」と分類することで、エージェントが新規 echo 追加時に迷わず日本語
  ベースを選択でき、既存実装の書き換えを誘発しない

### 5. ブランチ slug

**判断**: 本 Issue では確定させない。表中で「ブランチ名は英語固定（slug は ASCII lowercase
ハイフン区切り）」とだけ明記。slug の生成元（Issue タイトルを英語化 vs ローマ字化 vs その他）
は引き続き ad-hoc 運用とする。

**根拠**:

- requirements.md の Open Questions で「人間判断が必要な可能性あり」と明記されている
- 本 Issue は方針節の明文化が主目的であり、slug 生成規約の確定は別 Issue として切り出す
  方が責務分離が綺麗

## 確認事項（Reviewer / レビュワー判断ポイント）

- **配置位置の妥当性**: 言語方針節を `## 技術スタック` の前に置いたが、`## エージェント
  連携ルール` の周辺に置くべきかは判断分かれる可能性あり。本実装では「言語選択は複数節を
  横断する基本方針」と捉えて前方配置とした
- **bash ログ出力の分類**: 「混在許容」としたが、idd-claude メンテナとして「日本語ベース
  に統一すべき」と判断するなら別 Issue で書き換え対応する余地あり（本 Issue では Out of
  Scope）
- **slug の生成元**: 確定させていない。Issue タイトルを英語化するかローマ字化するかは
  ad-hoc のまま。確定が必要なら別 Issue を切る
- **README への参照**: requirements.md の Out of Scope に「README 本文の言語方針記述は
  CLAUDE.md とのリンクのみ追加可能性あり」とあるが、本実装では README には触れていない。
  追加が必要なら別途対応
